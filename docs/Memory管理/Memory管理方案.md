# Memory 管理方案

## 1. 目标

Memory 管理用于让 AI 在长会话中保持上下文，同时避免把所有历史消息无限塞进模型。

目标是：

- 保留最近对话的连续性
- 记住会话中的关键结论
- 记录文件访问和编辑历史
- 避免旧消息挤占文件内容上下文
- 用户可以查看和删除记忆

## 2. 核心原则

- 不无限发送全部历史消息
- 文件内容优先级高于旧聊天历史
- 用户主动保存的记忆优先级高于自动摘要
- 不自动记忆敏感信息
- Memory 必须可查看、可删除
- 结构化记录文件访问，不混在普通聊天文本里

## 3. Memory 分层

### 3.1 最近对话

保留最近 N 条消息，直接发送给模型。

当前可以先保留最近 20 条。

用途：

- 维持当前对话连续性
- 处理短期指代，比如“这个文件”“刚才那个问题”

### 3.2 会话摘要

当会话消息过多时，把较早的消息压缩成摘要。

发送给模型时使用：

```text
会话摘要 + 最近 20 条消息 + 当前问题
```

用途：

- 保留长期会话目标
- 保留已经达成的结论
- 减少 token 占用

### 3.3 文件访问记录

每一轮消息记录 AI 实际访问过的文件。

记录内容：

- 用户输入中定位到的路径
- 实际读取过的文件
- 拒绝读取的文件和原因
- 后续编辑过的文件

用途：

- 让历史会话可追溯
- 避免 AI 假装读过文件
- 后续做安全记录和审计

### 3.4 项目记忆

保存和当前项目长期相关的信息。

例如：

- 项目目标
- 常用目录
- 命名习惯
- 用户偏好
- 常见任务流程

项目记忆应该由用户主动保存，首版不建议自动写入。

## 4. 推荐发送结构

发送给模型时，建议按以下顺序组织：

```text
system:
  产品规则
  工具能力边界
  文件访问规则

memory:
  会话摘要
  项目记忆
  当前文件上下文

history:
  最近 20 条消息

user:
  当前用户问题
  本轮引用文件清单
```

## 5. 建议数据结构

### 5.1 会话摘要

```ts
type SessionMemory = {
  sessionId: string;
  summary: string;
  updatedAt: string;
};
```

### 5.2 文件访问记录

```ts
type MessageFileContext = {
  messageId: string;
  locatedPaths: string[];
  readFiles: string[];
  skippedFiles: Array<{
    path: string;
    reason: string;
  }>;
  editedFiles: string[];
};
```

### 5.3 项目记忆

```ts
type ProjectMemory = {
  id: string;
  scope: 'global' | 'workspace';
  workspacePath: string | null;
  content: string;
  source: 'user' | 'summary' | 'system';
  createdAt: string;
  updatedAt: string;
};
```

## 6. MVP 实现范围

第一阶段只做两件事：

- 会话摘要
- 文件访问记录

暂不做：

- 自动长期记忆
- 向量检索
- 跨项目记忆共享
- 自动记用户隐私信息

## 7. 会话摘要策略

建议规则：

- 最近 20 条消息直接保留
- 超出 20 条的旧消息进入摘要
- 摘要更新后，旧消息不再全部发送给模型
- 摘要内容需要简短、事实化

摘要应该包含：

- 当前任务目标
- 已完成事项
- 用户明确要求
- 重要决策
- 当前未解决问题

摘要不应该包含：

- 密码
- API Key
- 私钥
- 大段文件原文

## 8. 文件访问记录策略

每次发送消息后记录：

- 本轮定位了哪些路径
- 本轮读取了哪些文件
- 本轮跳过了哪些文件
- 后续如果编辑文件，也记录编辑结果

这些记录用于历史追溯，不一定每次都发送给模型。

## 9. UI 建议

后续可以增加一个 Memory 页面或弹窗。

展示：

- 当前会话摘要
- 本会话访问过的文件
- 项目记忆列表
- 删除记忆按钮
- 重新生成摘要按钮

## 10. 推荐实现顺序

1. 数据库增加 `session_memory` 表
2. 数据库增加 `message_file_context` 表
3. 每轮对话保存文件访问记录
4. 会话超过 20 条后生成摘要
5. 发送消息时附带摘要和最近 20 条消息
6. 增加查看摘要和文件访问记录的 UI
7. 后续再做项目记忆

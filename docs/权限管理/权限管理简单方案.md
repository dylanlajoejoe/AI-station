# 权限管理简单方案

## 1. 当前状态

当前阶段为了验证能力，文件访问先全量放开。

这意味着：

- AI 可以根据用户输入的绝对路径定位文件
- 引用文件可以从工作区外读取
- 隐藏文件和敏感文件暂不做强制拦截
- 文件仍然只读，不允许自动写入

这个状态适合开发验证，不适合正式发布。

## 2. 权限管理目标

后续需要把文件访问改成可控授权。

目标是：

- 默认安全
- 用户能主动放开权限
- AI 不能绕过用户授权
- 用户能看到当前授权了哪些目录
- 用户可以随时移除授权

## 3. 最简单的权限模型

只做两类权限：

- 当前工作区权限
- 临时外部路径权限

### 3.1 当前工作区权限

用户选择目录后，该目录成为当前工作区。

AI 默认可以：

- 查看目录结构
- 定位工作区内文件
- 读取用户引用的工作区内文本文件

AI 默认不可以：

- 修改文件
- 删除文件
- 读取未引用文件内容

### 3.2 临时外部路径权限

当用户输入工作区外路径时，系统弹窗询问：

```text
该路径不在当前工作区内，是否允许本次访问？
路径：D:\example\test.txt
```

用户确认后，只允许本次请求访问该路径。

用户取消后，不访问该路径。

## 4. 权限等级

首版可以只做三个等级：

### 4.1 查看目录

允许 AI 查看目录结构。

不包含文件内容。

### 4.2 读取文件

允许 AI 读取用户引用的文件内容。

只支持文本文件。

### 4.3 编辑文件

允许 AI 生成修改方案。

真正写入前必须用户确认。

## 5. 建议交互

### 5.1 外部路径访问

1. 用户在对话框输入外部路径
2. 系统识别路径不在当前工作区
3. 弹窗询问是否允许本次访问
4. 用户确认后继续定位或读取
5. 用户取消后返回拒绝提示

### 5.2 敏感路径访问

敏感路径包括：

- `.git`
- `.env`
- `credentials.json`
- `secrets.json`
- SSH 私钥文件

访问敏感路径时也弹窗确认。

提示文案要更明确：

```text
该路径可能包含敏感信息，是否允许本次读取？
```

## 6. 数据结构

```ts
type PermissionScope = {
  id: string;
  path: string;
  type: 'workspace' | 'temporary';
  permissions: Array<'list' | 'read' | 'edit'>;
  createdAt: string;
  expiresAt: string | null;
};
```

## 7. MVP 实现顺序

1. 恢复工作区边界校验
2. 外部路径访问前弹窗确认
3. 确认后只对本次请求放行
4. 敏感路径访问前弹窗确认
5. UI 展示当前授权路径
6. 支持移除授权路径

## 8. 发布前必须补齐

- 不允许默认全盘访问
- 不允许 AI 静默读取敏感文件
- 不允许 AI 自动写文件
- 文件写入必须用户确认
- 权限记录要能展示给用户

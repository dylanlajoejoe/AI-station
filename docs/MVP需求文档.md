# MVP 需求文档

## 1. MVP 目标

做一个最小可用版本，让用户可以进入本地目录、查看文件树，并和 AI 对话。

文件树里的 `添加到对话` 只负责把文件或文件夹全名插入输入框，省去用户复制粘贴名称的操作。

## 2. MVP 做什么

### 2.1 进入目录

- 用户可以选择一个本地目录
- 页面显示当前目录名称或路径

### 2.2 查看文件树

- 显示当前目录下的文件和文件夹
- 支持展开和折叠文件夹

### 2.3 添加到对话

- 用户可以右键文件或文件夹
- 右键菜单显示 `添加到对话`
- 点击后，把该文件或文件夹全名插入到对话输入框
- 用户可以继续编辑输入框内容

说明：

- 不读取文件内容
- 不上传文件
- 不把文件作为附件
- 不维护文件上下文
- 不做复杂引用系统

### 2.4 AI 对话

- 用户可以输入问题并发送
- AI 根据输入框文字回答
- 如果输入框里有文件名，AI 也只是把它当普通文字理解

### 2.5 历史会话

- 用户可以看到历史会话列表
- 用户可以继续打开历史会话

## 3. MVP 不做什么

- 不读取本地文件内容
- 不上传文件
- 不做文件总结
- 不做多文件对比
- 不做文件重命名
- 不做文件删除
- 不做复制路径
- 不做系统打开文件
- 不做文件类型识别
- 不做知识库
- 不做 Agent 自动执行

## 4. 页面

### 4.1 工作区页面

页面包含三部分：

- 左侧：文件树
- 中间或顶部：当前目录信息
- 右侧或主体：AI 对话区

### 4.2 历史会话页面

页面包含：

- 历史会话列表
- 会话标题
- 最近更新时间

## 5. 基础流程

1. 用户打开产品
2. 用户选择本地目录
3. 页面显示目录文件树
4. 用户右键某个文件或文件夹
5. 用户点击 `添加到对话`
6. 文件或文件夹全名出现在输入框里
7. 用户补充问题并发送
8. AI 根据输入框文字回答

## 6. 验收标准

- 可以选择本地目录
- 可以显示目录文件树
- 可以展开和折叠文件夹
- 可以右键文件或文件夹
- 可以点击 `添加到对话`
- 点击后文件或文件夹全名进入输入框
- 可以正常发送对话
- AI 可以正常返回回答
- 可以看到历史会话列表
- 不会读取本地文件内容

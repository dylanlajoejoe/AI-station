# 前端方案

## 1. 产品界面目标

「轻量级 AI 工作区连接器」采用 Electron 桌面应用形态。前端运行在 Electron Renderer 中，界面要像一个熟悉的本地文件管理器，而不是网盘后台或知识库后台。

当前可在 Linux 环境开发前端，但最终界面实现和验收以 Windows 桌面端为准。布局、路径展示、系统目录入口和窗口尺寸都要优先适配 Windows 使用习惯。

用户的主要感受应该是：

- 我能选择一个本地目录
- 我能浏览目录里的文件和文件夹
- 我能把文件名或文件夹名快速添加到对话输入框
- 我能在右侧和 AI 对话
- 我能重新打开历史会话

## 2. 技术栈

- React
- TypeScript
- Vite
- React Router
- Zustand
- CSS Modules 或普通 CSS
- Electron Renderer
- Electron Preload API

说明：

- 前端不按纯网页能力设计
- 本地目录选择、目录读取、AI 调用都通过 Electron Preload API 完成
- 前端不直接使用 Node.js、`fs`、`path` 等本地能力
- 路径展示必须兼容 Windows 盘符、反斜杠、中文目录和空格目录

## 3. 信息架构

主界面分为 4 个区域：

```text
+--------------------------------------------------------------------------------+
| 顶部工具栏：产品名 / 当前目录路径                                                |
+--------------------+--------------------------------------+--------------------+
| 左侧文件目录        | 中间文件信息区                        | 右侧 AI 助手面板    |
| 选择目录 / 文件树   | 当前选中文件或文件夹信息               | 对话 / 输入框        |
+--------------------+--------------------------------------+--------------------+
| 底部状态栏：基础状态 / 错误提示                                                  |
+--------------------------------------------------------------------------------+
```

## 4. 页面布局

### 4.1 顶部工具栏

用途：让用户知道当前打开的是哪个工作区。

包含：

- 产品名：轻量级 AI 工作区连接器
- 当前目录路径
- 基础状态提示

视觉建议：

- 高度约 52px
- 背景使用暖灰白或浅雾蓝
- 当前路径比产品名更突出

### 4.2 左侧文件目录

用途：展示用户选择的本地目录，并提供目录选择入口。

内容：

- `选择目录` 按钮
- 当前目录路径
- 当前目录下的文件和文件夹树
- 文件或文件夹右键菜单

空状态：

- 用户未选择目录时，显示 `请选择一个本地目录`
- `选择目录` 按钮仍显示在左侧顶部

交互：

- 点击 `选择目录` 后打开系统目录选择窗口
- 点击文件夹后展开或折叠
- 右键文件或文件夹显示 `添加到对话`
- 点击 `添加到对话` 后，将文件或文件夹全名插入右侧输入框

视觉建议：

- 宽度 240px 到 280px
- 保持轻量，不做复杂卡片
- 图标简单清晰
- 当前选中项使用浅蓝灰背景

### 4.3 中间文件信息区

用途：展示当前选中文件或文件夹的基础信息。

展示内容：

- 文件名或文件夹名
- 类型
- 大小
- 修改时间
- 路径

状态示例：

- `未选择文件`
- `已选择文件`
- `已选择文件夹`
- `无法读取目录结构`

MVP 当前阶段：

- 显示文件和文件夹基础信息
- 不读取文件真实内容
- 不做内容预览

MVP 不做：

- 文件重命名
- 文件删除
- 文件移动
- 系统打开
- 文件编辑
- 文件内容预览

### 4.4 右侧 AI 助手面板

用途：提供 AI 对话，但不抢占文件浏览主体。

结构：

```text
+--------------------------+
| 对话消息列表              |
| 用户问题                  |
| AI 回复                   |
+--------------------------+
| 输入框                    |
| 发送按钮                  |
+--------------------------+
```

内容：

- 对话消息
- 输入框
- 发送按钮
- 历史会话入口

交互：

- 用户输入问题并发送
- AI 回复后追加到消息列表
- 从文件树点击 `添加到对话` 后，输入框追加对应名称
- 用户可以继续编辑输入框内容

视觉建议：

- 宽度 360px 到 420px
- 背景比文件区略深一点
- 对话气泡保持克制
- 输入框保持明显但不过度抢占空间

### 4.5 底部状态栏

用途：低打扰地表达基础状态。

内容：

- 当前是否已选择目录
- 当前目录读取是否成功
- AI 接口是否可用
- 最近一次错误提示

## 5. 关键组件

### 5.1 FolderTree

负责左侧文件树。

Props：

```ts
type FolderTreeProps = {
  rootPath: string | null;
  nodes: FileTreeNode[];
  onSelectDirectory: () => void;
  onToggleFolder: (path: string) => void;
  onAddToChat: (name: string) => void;
};
```

### 5.2 FileInfoPanel

负责中间文件信息区。

Props：

```ts
type FileInfoPanelProps = {
  selectedItem: FileTreeNode | null;
};
```

### 5.3 AiAssistantPanel

负责右侧 AI 面板。

Props：

```ts
type AiAssistantPanelProps = {
  messages: ChatMessage[];
  inputValue: string;
  onInputChange: (value: string) => void;
  onSendMessage: () => void;
};
```

### 5.4 HistoryList

负责历史会话列表。

Props：

```ts
type HistoryListProps = {
  sessions: ChatSession[];
  onOpenSession: (sessionId: string) => void;
};
```

## 6. 数据结构

### 6.1 文件树节点

```ts
type FileTreeNode = {
  id: string;
  name: string;
  path: string;
  kind: 'file' | 'folder';
  sizeText?: string;
  modifiedAtText?: string;
  children?: FileTreeNode[];
  isExpanded?: boolean;
};
```

### 6.2 对话消息

```ts
type ChatMessage = {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  createdAt: string;
};
```

### 6.3 历史会话

```ts
type ChatSession = {
  id: string;
  title: string;
  updatedAt: string;
};
```

## 7. 必须覆盖的关键状态

### 7.1 未选择目录

界面表现：

- 左侧显示空状态
- 中间显示引导说明
- 右侧 AI 面板可输入普通问题

### 7.2 已选择目录

界面表现：

- 左侧显示文件树
- 顶部显示当前目录路径
- 中间显示当前选中文件或文件夹信息

### 7.3 添加到对话

界面表现：

- 用户右键文件或文件夹
- 菜单显示 `添加到对话`
- 点击后输入框追加该文件或文件夹全名
- 用户可以继续编辑输入框

### 7.4 用户向 AI 提问

界面表现：

- 用户消息出现在 AI 面板
- AI 回复显示在消息列表
- 失败时展示错误提示

### 7.5 历史会话

界面表现：

- 用户可以看到历史会话列表
- 用户可以打开历史会话
- 会话消息可以恢复显示

## 8. 视觉风格

整体方向：普通文件管理器 + 克制 AI 助手。

颜色建议：

- 背景：暖灰白 `#F4F1EA`
- 面板：雾蓝灰 `#EAF0F2`
- 主色：深海蓝 `#174A5C`
- 文本：石墨灰 `#2F3437`
- 提醒色：琥珀色 `#B7791F`
- 边框：浅灰蓝 `#D5DEE2`

风格原则：

- 不使用 AI 紫色
- 不做赛博科技感
- 不做复杂仪表盘
- 文件浏览区域优先，AI 面板次之

## 9. 响应式设计

Electron 桌面端优先：

- 三栏布局
- 左侧 240px 到 280px
- 右侧 360px 到 420px
- 中间文件信息区自适应

窄屏兼容：

- 默认显示文件树和 AI 面板
- 中间文件信息区可压缩
- AI 面板通过侧边按钮打开

说明：

- MVP 主要适配桌面窗口
- 窄屏只保证可用，不作为主要体验目标

Windows 桌面端测试重点：

- 1366x768 小屏窗口
- 1920x1080 常规窗口
- 高 DPI 缩放
- 长路径和中文路径显示

## 10. 前端接口约定

前端通过 Electron Preload API 获取数据。

```ts
type FrontendApi = {
  selectDirectory: () => Promise<{ path: string } | null>;
  listDirectory: (path: string) => Promise<FileTreeNode[]>;
  sendMessage: (params: SendMessageParams) => Promise<SendMessageResult>;
  listSessions: () => Promise<ChatSession[]>;
  openSession: (sessionId: string) => Promise<ChatMessage[]>;
};
```

## 11. MVP 前端不做

- 不做复杂后台管理
- 不做网盘式上传下载
- 不做文件编辑
- 不做文件删除
- 不做文件内容预览
- 不做图表仪表盘
- 不做知识库管理

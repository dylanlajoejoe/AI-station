# 前端方案

## 1. 产品界面目标

「轻量级 AI 工作区连接器」采用 Electron 桌面应用形态。前端运行在 Electron Renderer 中，界面要像一个熟悉的本地文件管理器，而不是权限后台、网盘后台或知识库后台。

当前可在 Linux 环境开发前端，但最终界面实现和验收以 Windows 桌面端为准。布局、路径展示、系统目录入口和窗口尺寸都要优先适配 Windows 使用习惯。

用户的主要感受应该是：

- 我能像平时一样浏览文件
- AI 在旁边辅助我理解资料
- AI 不能随便读文件
- 需要额外权限时，我能清楚决定是否授权
- 每次真正读取文件前，我都能看到清单

## 2. 技术栈

- React
- TypeScript
- Vite
- React Router
- Zustand
- CSS Modules 或普通 CSS
- Electron Renderer
- Electron Preload API

说明：

- 前端不按纯网页能力设计
- 本地文件访问、目录读取、AI 调用都通过 Electron Preload 暴露的安全 API 完成
- 前端不直接使用 Node.js、`fs`、`path` 等本地能力
- 路径展示必须兼容 Windows 盘符、反斜杠、中文目录和空格目录

## 3. 信息架构

主界面分为 5 个区域：

```text
+--------------------------------------------------------------------------------+
| 顶部工具栏：路径面包屑 / 搜索 / 视图切换 / 当前权限状态                         |
+--------------------+--------------------------------------+--------------------+
| 左侧文件夹树        | 中间文件列表                          | 右侧 AI 助手面板    |
| 常用目录入口        | 文件名 / 类型 / 修改时间 / 大小 / 状态 | 当前引用 / 对话 / 输入 |
| 文件夹层级          | 列表视图或网格视图                     | 授权提醒 / 读取提示 |
+--------------------+--------------------------------------+--------------------+
| 底部状态栏：只读模式 / 本次会话已读取 X 个文件 / 当前路径状态                  |
+--------------------------------------------------------------------------------+
```

## 4. 页面布局

### 4.1 顶部工具栏

用途：让用户知道自己在哪里，以及当前权限状态。

包含：

- 产品名：轻量级 AI 工作区连接器
- 路径面包屑：例如 `文档 / 客户资料 / 2024 合同`
- 搜索框：搜索当前目录内文件名
- 视图切换：列表 / 网格
- 权限状态：`只读模式`、`当前目录可访问`

视觉建议：

- 高度约 52px
- 背景使用暖灰白或浅雾蓝
- 路径面包屑比产品名更突出
- 权限状态使用小标签，不要做成警告横幅

### 4.2 左侧文件夹树

用途：提供熟悉的文件管理器入口。

内容：

- 桌面
- 下载
- 文档
- 图片
- 项目
- 最近访问
- 用户添加过的目录

Windows 展示时应对应常见入口：

- 桌面
- 下载
- 文档
- 图片
- 此电脑
- 最近访问

交互：

- 点击目录后，中间区域显示该目录内容
- 可展开和折叠文件夹
- 当前目录高亮
- 无权限目录显示浅灰锁标识

视觉建议：

- 宽度 240px 到 280px
- 保持轻量，不做复杂卡片
- 图标简单清晰
- 当前选中项使用浅蓝灰背景

### 4.3 中间文件列表

用途：主工作区，展示用户当前目录下的文件和文件夹。

列表字段：

- 文件名
- 类型
- 修改时间
- 大小
- 状态

状态示例：

- `可访问`
- `未授权`
- `已选择`
- `风险提示`

交互：

- 点击文件夹进入下一级
- 点击文件选中或打开摘要预览
- 多选文件
- 右键文件显示轻量菜单
- 支持列表视图和网格视图

右键菜单：

- 选择给 AI
- 预览摘要
- 查看信息

MVP 不做：

- 文件重命名
- 文件删除
- 文件移动
- 系统打开

### 4.4 右侧 AI 助手面板

用途：提供 AI 对话，但不抢占文件浏览主体。

结构：

```text
+--------------------------+
| 当前引用文件              |
| file-a.pdf  file-b.docx  |
+--------------------------+
| 对话消息列表              |
| 用户问题                  |
| AI 回复                   |
| 授权请求卡片              |
| 读取确认卡片              |
+--------------------------+
| 输入框                    |
| 发送按钮                  |
+--------------------------+
```

内容：

- 当前引用文件
- AI 正在尝试访问的上下文
- 对话消息
- 输入框
- 发送按钮

视觉建议：

- 宽度 360px 到 420px
- 背景比文件区略深一点
- 对话气泡保持克制
- 当前引用文件使用小标签展示
- 授权请求和读取确认使用卡片嵌入对话流

### 4.5 底部状态栏

用途：低打扰地表达安全状态。

内容：

- `只读模式`
- `本次会话已读取 X 个文件`
- `当前目录：可访问`
- `AI 读取记录`

交互：

- 点击 `AI 读取记录` 打开侧边抽屉
- 状态栏不抢主界面注意力

## 5. 关键组件

### 5.1 FolderTree

负责左侧文件夹树。

Props：

```ts
type FolderTreeProps = {
  roots: FolderNode[];
  currentPath: string | null;
  onSelectDirectory: (path: string) => void;
};
```

### 5.2 FileList

负责中间文件列表。

Props：

```ts
type FileListProps = {
  files: FileItem[];
  selectedFileIds: string[];
  viewMode: 'list' | 'grid';
  onOpenDirectory: (path: string) => void;
  onSelectFile: (fileId: string) => void;
  onPreviewFile: (fileId: string) => void;
};
```

### 5.3 AiAssistantPanel

负责右侧 AI 面板。

Props：

```ts
type AiAssistantPanelProps = {
  contextFiles: FileItem[];
  messages: ChatMessage[];
  pendingAccessRequest?: AccessRequest;
  pendingReadConfirm?: ReadConfirm;
  onSendMessage: (content: string) => void;
  onApproveAccess: (requestId: string) => void;
  onDenyAccess: (requestId: string) => void;
  onSelectPartialFiles: (requestId: string) => void;
};
```

### 5.4 AccessRequestCard

用于展示 AI 权限外访问请求。

内容：

- 标题：AI 需要访问额外文件夹
- 访问目录
- 访问原因
- 访问方式：只读
- 授权范围：本次会话有效 / 持续允许
- 操作：允许本次访问、选择部分文件、拒绝

### 5.5 ReadConfirmDialog

用于读取文件前确认。

内容：

- 即将读取的文件清单
- 文件来源目录
- 是否只读
- 敏感风险提醒
- 操作：确认读取、取消

### 5.6 ReadHistoryDrawer

用于查看本次会话读取记录。

内容：

- 读取时间
- 文件名
- 来源目录
- 触发原因
- 读取结果状态

## 6. 数据结构

### 6.1 文件夹节点

```ts
type FolderNode = {
  id: string;
  name: string;
  path: string;
  isAccessible: boolean;
  isReadOnly: boolean;
  children?: FolderNode[];
};
```

### 6.2 文件项

```ts
type FileItem = {
  id: string;
  name: string;
  path: string;
  directoryPath: string;
  type: 'folder' | 'pdf' | 'docx' | 'xlsx' | 'txt' | 'md' | 'csv' | 'unknown';
  sizeText: string;
  modifiedAtText: string;
  accessStatus: 'accessible' | 'unauthorized' | 'selected' | 'risk';
  riskLevel?: 'none' | 'low' | 'medium' | 'high';
};
```

### 6.3 授权请求

```ts
type AccessRequest = {
  id: string;
  directoryPath: string;
  reason: string;
  accessMode: 'read_only';
  scope: 'current_session' | 'persistent';
  status: 'pending' | 'approved' | 'denied' | 'partial';
};
```

### 6.4 读取确认

```ts
type ReadConfirm = {
  id: string;
  files: FileItem[];
  reason: string;
  accessMode: 'read_only';
  riskSummary: string | null;
};
```

### 6.5 读取记录

```ts
type ReadLog = {
  id: string;
  fileName: string;
  filePath: string;
  directoryPath: string;
  reason: string;
  readAt: string;
  status: 'success' | 'failed' | 'canceled';
};
```

## 7. 必须覆盖的关键状态

### 7.1 普通文件浏览状态

界面表现：

- 左侧显示常用目录和文件夹树
- 中间显示当前目录文件列表
- 右侧 AI 面板显示空状态或历史对话
- 底部显示 `只读模式`

### 7.2 用户向 AI 提问

界面表现：

- 用户消息出现在 AI 面板
- 当前选中文件显示在引用区
- AI 回复显示在消息列表
- 如果文件上下文足够，直接回答

### 7.3 AI 发起权限外访问请求

界面表现：

- AI 面板内出现 `AccessRequestCard`
- 弹窗或卡片说明访问原因和目录
- 主界面可继续浏览，但发送动作等待用户决定

### 7.4 用户同意授权

界面表现：

- 请求卡片状态变为 `已允许`
- 目录状态从 `未授权` 变为 `可访问`
- 接下来进入读取文件前确认

### 7.5 用户拒绝授权

界面表现：

- 请求卡片状态变为 `已拒绝`
- AI 给出受限回答
- 不读取权限外文件

### 7.6 AI 读取文件前确认

界面表现：

- 弹出 `ReadConfirmDialog`
- 列出将读取的文件
- 显示来源目录
- 显示只读说明
- 显示风险提示
- 用户确认后才读取

### 7.7 用户查看读取记录

界面表现：

- 点击底部 `AI 读取记录`
- 右侧或底部打开抽屉
- 按时间线展示读取过的文件

## 8. 授权请求弹窗文案

标题：

```text
AI 需要访问额外文件夹
```

说明：

```text
为了回答你的问题，AI 可能需要读取以下目录中的文件。
```

字段：

```text
目录路径：/Documents/客户资料/2024 合同
访问原因：你提到了去年合同，但当前目录中没有相关文件
访问方式：只读
授权范围：本次会话有效
```

按钮：

```text
允许本次访问
选择部分文件
拒绝
```

## 9. 视觉风格

整体方向：普通文件管理器 + 克制 AI 助手。

颜色建议：

- 背景：暖灰白 `#F4F1EA`
- 面板：雾蓝灰 `#EAF0F2`
- 主色：深海蓝 `#174A5C`
- 文本：石墨灰 `#2F3437`
- 安全状态：墨绿色 `#2F6B4F`
- 提醒色：琥珀色 `#B7791F`
- 边框：浅灰蓝 `#D5DEE2`

风格原则：

- 不使用 AI 紫色
- 不做赛博科技感
- 不做复杂仪表盘
- 不把权限管理做成主界面
- 文件浏览区域优先，AI 面板次之
- 权限请求明显但不恐吓用户

## 10. 响应式设计

Electron 桌面端优先：

- 三栏布局
- 左侧 240px 到 280px
- 右侧 360px 到 420px
- 中间文件区自适应

窄屏兼容：

- 默认显示文件列表
- 文件夹树收进抽屉
- AI 面板通过侧边按钮打开
- 授权请求和读取确认使用居中弹窗

说明：

- MVP 主要适配桌面窗口
- 窄屏只保证可用，不作为主要体验目标

Windows 桌面端测试重点：

- 1366x768 小屏窗口
- 1920x1080 常规窗口
- 高 DPI 缩放
- 长路径和中文路径显示

## 11. 前端接口约定

前端通过 Electron Preload API 获取数据。

```ts
type FrontendApi = {
  listRoots: () => Promise<FolderNode[]>;
  listDirectory: (path: string) => Promise<FileItem[]>;
  sendMessage: (params: SendMessageParams) => Promise<SendMessageResult>;
  approveAccess: (requestId: string, scope: AccessRequest['scope']) => Promise<void>;
  denyAccess: (requestId: string) => Promise<void>;
  confirmRead: (confirmId: string) => Promise<void>;
  cancelRead: (confirmId: string) => Promise<void>;
  listReadLogs: (sessionId: string) => Promise<ReadLog[]>;
};
```

## 12. MVP 前端不做

- 不做复杂后台管理
- 不做网盘式上传下载
- 不做文件编辑
- 不做文件删除
- 不做图表仪表盘
- 不做知识库管理
- 不做复杂权限规则配置

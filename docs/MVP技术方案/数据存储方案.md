# 数据存储方案

## 1. 技术栈

推荐：

- SQLite
- better-sqlite3
- Electron app data 目录

备选：

- 本地 JSON 文件

MVP 推荐 SQLite，因为会话和消息天然适合表结构，后续扩展成本更低。

当前可在 Linux 环境开发存储逻辑，但最终数据存储必须在 Windows Electron 应用中稳定运行。数据库路径、文件权限和打包后的读写能力都以 Windows 为主要验收标准。

## 2. 数据库安装方式

Windows 用户不需要单独安装数据库软件。

MVP 使用内嵌式 SQLite：

- 不安装 MySQL
- 不安装 PostgreSQL
- 不要求用户配置数据库服务
- 不启动独立数据库进程
- 应用启动后自动创建本地 `.sqlite` 数据库文件
- 应用通过 Node.js SQLite 驱动直接读写该文件

对用户来说，数据库只是应用内部数据文件，不是额外依赖。

## 3. 存储位置

数据库文件存放在应用数据目录。

示例：

```text
userData/
  ai-workstation.sqlite
```

Electron 中可通过 `app.getPath('userData')` 获取目录。

Windows 示例路径：

```text
C:\\Users\\用户名\\AppData\\Roaming\\轻量级 AI 工作区连接器\\ai-workstation.sqlite
```

## 4. 存储内容

保存：

- 会话信息
- 会话消息
- 当前会话所属目录路径
- AI 配置基础信息

不保存：

- 文件内容
- 文件索引
- 完整目录树快照
- 附件信息

## 5. 数据表设计

### 5.1 sessions

```sql
CREATE TABLE sessions (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  workspace_path TEXT,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);
```

字段说明：

- `id`：会话 ID
- `title`：会话标题，可取用户第一条消息前 20 个字符
- `workspace_path`：会话创建时所在目录
- `created_at`：创建时间
- `updated_at`：更新时间

### 5.2 messages

```sql
CREATE TABLE messages (
  id TEXT PRIMARY KEY,
  session_id TEXT NOT NULL,
  role TEXT NOT NULL,
  content TEXT NOT NULL,
  created_at TEXT NOT NULL,
  FOREIGN KEY (session_id) REFERENCES sessions(id)
);
```

字段说明：

- `id`：消息 ID
- `session_id`：所属会话 ID
- `role`：`user` 或 `assistant`
- `content`：消息正文
- `created_at`：创建时间

### 5.3 settings

```sql
CREATE TABLE settings (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL,
  updated_at TEXT NOT NULL
);
```

可保存：

- AI Base URL
- AI Model
- 最近打开目录

API Key 是否保存需要单独评估。MVP 可以先不提供界面保存 API Key，优先使用环境变量。

## 6. 数据访问接口

### 6.1 创建会话

```ts
type CreateSessionInput = {
  workspacePath: string | null;
};
```

返回：

```ts
type Session = {
  id: string;
  title: string;
  workspacePath: string | null;
  createdAt: string;
  updatedAt: string;
};
```

### 6.2 保存消息

```ts
type CreateMessageInput = {
  sessionId: string;
  role: 'user' | 'assistant';
  content: string;
};
```

### 6.3 查询会话列表

按 `updated_at` 倒序返回。

```ts
type ListSessionsResult = Session[];
```

### 6.4 查询会话详情

```ts
type SessionDetail = {
  session: Session;
  messages: ChatMessage[];
};
```

## 7. 会话标题规则

MVP 简单规则：

- 新建会话默认标题为 `新会话`
- 用户发送第一条消息后，用第一条用户消息生成标题
- 标题最多 20 个字符
- 换行替换为空格

## 8. 数据迁移

MVP 可准备简单版本号：

```text
schema_version = 1
```

后续升级时根据版本执行迁移 SQL。

## 9. 风险

- SQLite 原生依赖打包可能需要额外配置
- 数据库文件损坏需要兜底处理
- API Key 保存涉及安全问题
- JSON 方案前期简单但后续查询和迁移麻烦
- Windows 打包后数据库路径和写入权限需要重点测试

## 10. Windows 兼容要求

- 数据库必须存放在 `app.getPath('userData')`
- 不把数据库写入安装目录
- 不要求 Windows 用户安装任何数据库
- 支持中文 Windows 用户名路径
- 支持路径中包含空格
- 打包后仍能创建、读取、更新数据库
- Windows 真机测试必须覆盖新建会话、保存消息、读取历史会话

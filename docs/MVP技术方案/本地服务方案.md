# 本地服务方案

## 1. 技术栈

- Electron Main Process
- Node.js
- TypeScript
- fs/promises
- path
- ipcMain

当前可在 Linux 服务器开发，但最终本地服务必须在 Windows 上稳定运行。路径、目录权限、系统目录选择、数据库位置都以 Windows 兼容为优先。

## 2. 本地服务职责

本地服务运行在 Electron 主进程中，负责前端不能直接做的本地能力。

主要职责：

- 打开系统目录选择窗口
- 读取目录结构
- 返回文件和文件夹列表
- 调用 AI 服务
- 保存和读取会话数据
- 处理本地错误

Windows 重点职责：

- 正确处理盘符路径，例如 `C:\\Users\\name\\Documents`
- 正确处理中文目录和带空格目录
- 正确处理 Windows 无权限目录
- 正确使用 `app.getPath` 获取系统目录

## 4. IPC 接口

### 4.1 选择目录

接口名：`dialog:selectDirectory`

请求：无

响应：

```ts
type SelectDirectoryResult = {
  canceled: boolean;
  path: string | null;
};
```

说明：

- 使用 Electron `dialog.showOpenDialog`
- 只允许选择目录
- 用户取消时返回 `canceled: true`
- Windows 下使用系统原生目录选择体验

### 4.2 读取目录树

接口名：`fileTree:list`

请求：

```ts
type ListFileTreeParams = {
  directoryPath: string;
};
```

响应：

```ts
type FileTreeNode = {
  id: string;
  name: string;
  path: string;
  type: 'file' | 'directory';
  hasChildren?: boolean;
};
```

说明：

- 只读取指定目录第一层
- 不递归读取整个目录
- 不读取文件内容
- `id` 可直接使用路径或路径 hash
- 路径处理必须使用 Node `path` API

### 4.3 发送消息

接口名：`chat:sendMessage`

请求：

```ts
type SendMessageParams = {
  sessionId: string;
  content: string;
};
```

响应：

```ts
type SendMessageResult = {
  userMessage: ChatMessage;
  assistantMessage: ChatMessage;
};
```

说明：

- 保存用户消息
- 调用 AI 接口
- 保存助手消息
- 返回两条消息

### 4.4 会话列表

接口名：`session:list`

请求：无

响应：

```ts
type ListSessionsResult = Session[];
```

### 4.5 获取会话详情

接口名：`session:get`

请求：

```ts
type GetSessionParams = {
  sessionId: string;
};
```

响应：

```ts
type SessionDetail = {
  session: Session;
  messages: ChatMessage[];
};
```

### 4.6 创建会话

接口名：`session:create`

请求：

```ts
type CreateSessionParams = {
  workspacePath: string | null;
};
```

响应：

```ts
type CreateSessionResult = Session;
```

## 5. 目录读取规则

目录读取只允许做这些事：

- 读取目录项名称
- 判断目录项是文件还是文件夹
- 返回路径
- 返回是否可能有子节点

禁止做这些事：

- 读取文件内容
- 递归扫描整个目录
- 上传文件
- 修改文件
- 删除文件

## 6. 错误处理

常见错误：

- 目录不存在
- 目录无权限
- 目录读取失败
- 路径格式异常
- AI 接口失败
- 数据库读写失败

统一错误结构：

```ts
type AppError = {
  code: string;
  message: string;
};
```

建议错误码：

- `DIRECTORY_NOT_FOUND`
- `DIRECTORY_PERMISSION_DENIED`
- `DIRECTORY_READ_FAILED`
- `AI_REQUEST_FAILED`
- `SESSION_NOT_FOUND`
- `DATABASE_ERROR`

## 7. 安全边界

- Renderer 不直接使用 Node.js API
- 只通过 preload 暴露白名单 API
- `contextIsolation` 开启
- `nodeIntegration` 关闭
- 不暴露任意路径读取接口
- 目录读取接口只返回目录项信息

## 8. 本地服务不做

- 不启动独立 HTTP 端口
- 不做文件内容读取
- 不做文件索引
- 不做文件搜索
- 不做文件修改
- 不做系统文件操作

## 9. Windows 兼容要求

- 不手动拼接路径分隔符
- 使用 `path.join`、`path.resolve`、`path.normalize`
- 支持盘符路径
- 支持中文路径
- 支持带空格路径
- 遇到无权限目录时返回错误，不崩溃
- 使用 `app.getPath('userData')` 保存应用数据
- Windows 真机测试必须覆盖目录选择、目录读取、会话保存和 AI 调用

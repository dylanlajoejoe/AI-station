# 技术方案

## 1. 目标

用轻量方式实现 MVP：选择本地目录、显示文件树、右键插入文件名、普通 AI 对话、保存历史会话。

## 2. 大致技术栈

- 前端：React + TypeScript
- 桌面壳：Electron
- 本地服务：Node.js
- AI 调用：OpenAI 兼容接口
- 本地存储：SQLite 或本地 JSON
- 构建工具：Vite

## 3. 开发部分划分

- 前端：见 `docs/前端方案.md`
- 本地服务：见 `docs/本地服务方案.md`
- AI 调用：见 `docs/AI调用方案.md`
- 数据存储：见 `docs/数据存储方案.md`

## 4. MVP 不做

- 不读取文件内容
- 不上传文件
- 不做文件索引
- 不做知识库
- 不做复杂权限系统

## 5. 主要风险

- 不同系统的目录路径格式不同
- 大目录加载可能变慢
- AI 接口可能失败或超时
- 本地数据存储需要避免丢失
